# FeatureLiftBench Task Format

This document is the **canonical specification** for one benchmark task. Machine-readable schema: [`harness/featureliftbench/schemas/task_metadata.schema.json`](../harness/featureliftbench/schemas/task_metadata.schema.json). Directory checks: `featureliftbench validate-task`.

Human design notes (optional but recommended for new hard tasks): [`task_designs/TEMPLATE.md`](task_designs/TEMPLATE.md).

## Task identity

| Rule | Detail |
| --- | --- |
| Location | `benchmark/tasks/<task_id>/` |
| `task_id` | Must **equal the directory name** |
| Naming pattern | `<source>__<feature_slug>__<serial>` — e.g. `sqlparse__parse_format_core__001` |
| Language | `python` only (v0) |
| `difficulty` | `easy` \| `medium` \| `hard` — required on all current tasks |
| `tags` | Optional strings for reporting slices (not separate benchmarks) |

Multiple tasks may share the same `source.commit` (multi-task-per-repo). They differ by `feature`, `output`, and tests.

## Directory layout (required)

```text
benchmark/tasks/<task_id>/
  metadata.json
  requirements.lock          # may be empty; path must match environment.dependency_lock
  repo/                      # pinned upstream snapshot
  public_tests/              # visible to agent during development
  hidden_tests/              # used only at evaluation time
  evaluation/
    forbidden_imports.txt    # one forbidden import name per line
    oracle_manifest.json     # JSON; oracle closure manifest for maintenance
```

Validated by [`harness/featureliftbench/validate.py`](../harness/featureliftbench/validate.py).

**Not required:** `evaluation/migrated_tests.txt` (legacy README mention only).

### Oracle submission (local, gitignored)

Reference answers live outside the task directory:

```text
benchmark/submissions/<task_id>/oracle/
  pyproject.toml             # optional but recommended
  featurelifted/
    ...
```

Built with `python3 harness/scripts/build_oracle_submission.py benchmark/tasks/<task_id>/`.

## Agent workspace (generated by harness)

`run-agent` copies a **redacted** view into `experiments/.../workspace/`:

```text
workspace/
  repo/
  public_tests/
  requirements.lock
  metadata.json              # redacted; no hidden path, no scoring_reference
  TASK.md                    # prompt generated from metadata
  submission/                # agent writes the answer here
```

Redaction logic: [`harness/featureliftbench/agent_runner.py`](../harness/featureliftbench/agent_runner.py) (`redact_task_metadata`).

## Submission format (agent output)

```text
submission/
  pyproject.toml             # recommended
  featurelifted/             # package name from metadata.output.package (always featurelifted)
    __init__.py
    ...
```

Rules:

- Must be installable or importable under `submission/` alone.
- Must not import forbidden packages (see `environment.forbidden_imports` and `evaluation/forbidden_imports.txt`).
- Must not depend on the original upstream package at runtime.
- Must pass `public_tests/` and `hidden_tests/` after evaluation install.

## `metadata.json` fields

### Required

| Field | Purpose |
| --- | --- |
| `task_id` | Unique id; matches directory name |
| `language` | `"python"` |
| `source` | `name`, `url`, `commit`, `license` |
| `feature` | `name`, `description`, `source_entrypoints`, `included_behaviors`, `excluded_behaviors` |
| `entanglement` | `level`, `types`, `description`, `signals` |
| `output` | Expected API: `package`, `import`, `callable`, `signature` |
| `environment` | Runtime constraints (see below) |
| `tests` | `public`, `hidden`, `command` paths |

### Recommended

| Field | Purpose |
| --- | --- |
| `difficulty` | Primary difficulty axis |
| `tags` | e.g. `multi-task-repo`, `functional-discriminator` |
| `scoring_reference` | Copy-All / Oracle LOC bytes for manual comparison (**not used by evaluator**) |

### `entanglement.types` (enum)

`framework_coupling`, `config_environment_coupling`, `global_state_registry_coupling`, `resource_coupling`, `implicit_dependency_coupling`, `parser_state_coupling`, `data_model_coupling`, `third_party_dependency_coupling`, `legacy_vibe_clutter`

### `environment`

| Key | Meaning |
| --- | --- |
| `python` | Target Python version string (e.g. `"3.11"`) |
| `network` | Must be `false` |
| `timeout_seconds` | Eval timeout |
| `dependency_lock` | Relative path, usually `"requirements.lock"` |
| `allowed_dependencies` | PyPI names allowed in lock file |
| `forbidden_dependencies` | Must not appear in submission deps |
| `forbidden_imports` | Must not appear in submission source |

Allowed and forbidden dependency sets must not overlap.

### `tests`

| Key | Typical value |
| --- | --- |
| `public` | `"public_tests/"` |
| `hidden` | `"hidden_tests/"` |
| `command` | `"pytest"` |

## Scoring (same for every task)

```text
FunctionalGate = BuildPass × TestPass × OriginalImportPass
ExtractionRatio = SubmissionPythonLOC / SourceRepoPythonLOC
final_score = FunctionalGate × (1 - ExtractionRatio)   # when gate passes
```

Defined in [`harness/featureliftbench/scoring.py`](../harness/featureliftbench/scoring.py).

## Adding or changing a task

1. Create `benchmark/tasks/<task_id>/` with the layout above.
2. Write `metadata.json` matching the schema.
3. Add `public_tests/` and `hidden_tests/` (pytest).
4. Fill `evaluation/forbidden_imports.txt` and `oracle_manifest.json`.
5. Run `featureliftbench validate-task benchmark/tasks/<task_id>/`.
6. Build and verify Oracle: `harness/scripts/build_oracle_submission.py` + `featureliftbench eval ...`.
7. Add [`docs/task_designs/<task_id>.md`](task_designs/) for non-trivial tasks.
8. Update [`benchmark_tasks.md`](benchmark_tasks.md) catalog.

Do **not** introduce alternate task roots, schemas, or scoring tracks.

## Validation checklist

```bash
pip install -e .

featureliftbench validate-task benchmark/tasks/<task_id>/

# optional: oracle must pass
featureliftbench eval \
  benchmark/tasks/<task_id> \
  benchmark/submissions/<task_id>/oracle \
  --output experiments/validate-<task_id>
```
